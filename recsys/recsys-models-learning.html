Layer 1 — 特征交互基础
01

FM 因子分解机

Factorization Machines | Steffen Rendle, 2010
核心洞察：用低秩矩阵分解来近似交叉权重矩阵。即使特征 i 和特征 j 从未在训练数据中共现，只要各自与其他特征有过交互，就能通过隐向量 \(\mathbf{v}_i\) 和 \(\mathbf{v}_j\) 估计它们的交互强度。

问题：为什么 LR 不够？

LR 是线性模型，只能捕捉单个特征的独立贡献。真正有价值的信号藏在特征组合中 (如"年龄=25 AND 类目=数码")。直接枚举所有二阶组合会导致参数爆炸 (\(O(n^2)\))，且稀疏数据下绝大多数参数无法充分训练。

FM 的解法：隐向量分解

FM 模型公式：

$$\hat{y}(\mathbf{x}) = w_0 + \sum_{i=1}^{n} w_i x_i + \sum_{i=1}^{n}\sum_{j=i+1}^{n} \langle \mathbf{v}_i, \mathbf{v}_j \rangle \cdot x_i x_j$$

其中 \(\mathbf{v}_i \in \mathbb{R}^k\) 是每个特征的 k 维隐向量，\(\langle \mathbf{v}_i, \mathbf{v}_j \rangle = \sum_{f=1}^k v_{i,f} \cdot v_{j,f}\)

O(kn) 计算技巧

二阶交互项 \(\sum_{i

$$\sum_{i=1}^{n}\sum_{j=i+1}^{n}\langle \mathbf{v}_i, \mathbf{v}_j\rangle x_i x_j = \frac{1}{2}\sum_{f=1}^{k}\left[\left(\sum_{i=1}^{n}v_{i,f}x_i\right)^2 - \sum_{i=1}^{n}v_{i,f}^2 x_i^2\right]$$

复杂度从 \(O(kn^2)\) 降为 \(O(kn)\)，其中 \(k\) 通常取 8-256，使得 FM 在百万级特征上也能高效训练。

PyTorch 实现

Python (PyTorch)
import torch
import torch.nn as nn

class FactorizationMachine(nn.Module):
    """FM 二阶交互层 — O(kn) 实现"""
    def forward(self, x):
        # x: (batch, num_features, embed_dim)
        square_of_sum = torch.sum(x, dim=1) ** 2        # (batch, k)
        sum_of_square = torch.sum(x ** 2, dim=1)        # (batch, k)
        interaction = 0.5 * torch.sum(square_of_sum - sum_of_square, dim=1, keepdim=True)
        return interaction                              # (batch, 1)

class FMForCTR(nn.Module):
    def __init__(self, num_features, embed_dim, num_dense=0):
        super().__init__()
        self.embedding = nn.Embedding(num_features, embed_dim)
        self.linear = nn.Embedding(num_features, 1)
        self.bias = nn.Parameter(torch.zeros(1))
        self.fm = FactorizationMachine()

    def forward(self, x):
        # x: (batch, num_fields) — sparse feature indices
        embed = self.embedding(x)              # (batch, fields, k)
        linear_out = self.linear(x).sum(dim=1) # (batch, 1)
        fm_out = self.fm(embed)                # (batch, 1)
        logit = self.bias + linear_out + fm_out
        return torch.sigmoid(logit.squeeze(1))   # (batch,)

FM 的意义与局限

优势

自动学习二阶特征交叉，\(O(kn)\) 线性复杂度，稀疏数据友好（通过隐向量泛化），可作为推荐/排序/CTR 的通用框架

局限

仅建模二阶交互，无法捕捉更高阶模式；表达能力受限于隐向量维度 k；缺少非线性变换

Layer 2 — Deep + Wide 范式
02

Wide & Deep

Google, DLRS 2016 | Google Play 推荐系统
核心洞察：记忆 (Memorization) 和泛化 (Generalization) 是推荐系统的两个基本能力。Wide 部分通过交叉特征实现"记住"历史模式，Deep 部分通过 DNN 实现"举一反三"。两者联合训练 (Joint Training)，而非简单 Ensemble。

记忆 vs 泛化

维度Wide (记忆)Deep (泛化)
模型LR + 交叉积特征DNN + Embedding
优势精确记住"看过A买了B"的共现模式发现"从未见过但合理"的新组合
劣势无法泛化到未见过的特征组合可能过度泛化，推荐不相关物品
类比背课文（精确但死板）理解原理（灵活但可能错误）

交叉积变换 (Cross-Product Transformation)

$$\phi_k(\mathbf{x}) = \prod_{i=1}^{d} x_i^{c_{ki}}, \quad c_{ki} \in \{0, 1\}$$

例：\(\phi(\text{gender=M}, \text{lang=EN}) = 1\) 当且仅当 gender=Male AND language=English

联合训练 vs Ensemble

联合训练 (Joint Training)

Wide 和 Deep 同时反向传播更新参数。Wide 部分只需要补充少量交叉特征，因为 Deep 部分已经学到了大部分模式。

Ensemble (不采用)

两个独立模型分别训练再合并。每个模型都需要足够大才能独立工作，无法互补。

Python (PyTorch)
class WideAndDeep(nn.Module):
    def __init__(self, wide_dim, embed_dims, hidden_units, num_classes=1):
        super().__init__()
        # Wide: 线性模型
        self.wide = nn.Linear(wide_dim, num_classes)

        # Deep: Embedding + MLP
        self.embeddings = nn.ModuleList([
            nn.Embedding(num_cat, dim) for num_cat, dim in embed_dims
        ])
        total_embed = sum(dim for _, dim in embed_dims)
        layers = []
        in_dim = total_embed
        for h in hidden_units:
            layers += [nn.Linear(in_dim, h), nn.ReLU(), nn.BatchNorm1d(h)]
            in_dim = h
        layers.append(nn.Linear(in_dim, num_classes))
        self.deep = nn.Sequential(*layers)

    def forward(self, x_wide, x_deep_cat):
        # Wide path
        wide_out = self.wide(x_wide)
        # Deep path
        embeds = [emb(x_deep_cat[:, i]) for i, emb in enumerate(self.embeddings)]
        deep_in = torch.cat(embeds, dim=1)
        deep_out = self.deep(deep_in)
        # Joint
        return torch.sigmoid(wide_out + deep_out).squeeze(1)

DeepFM

Huawei, IJCAI 2017 | 消除人工特征工程
核心洞察：将 Wide & Deep 的 Wide 部分替换为 FM，并让 FM 和 DNN 共享同一个 Embedding 层。这样既保留了 FM 自动学习二阶交互的能力，又利用了 DNN 学习高阶非线性交互的能力——完全不需要任何人工特征工程。

DeepFM vs Wide & Deep 关键区别

维度Wide & DeepDeepFM
Wide 部分LR + 手工交叉特征FM (自动二阶交互)
Embedding 共享Wide 和 Deep 各自独立FM 和 Deep 共享同一套

Python (PyTorch)
class DeepFM(nn.Module):
    def __init__(self, field_dims, embed_dim, hidden_units):
        super().__init__()
        num_fields = len(field_dims)
        # 共享 Embedding
        self.embedding = nn.Embedding(sum(field_dims), embed_dim)
        self.offsets = torch.tensor([0] + list(field_dims[:-1])).cumsum(0)

        # FM: 一阶 + 二阶
        self.linear = nn.Embedding(sum(field_dims), 1)
        self.fm = FactorizationMachine()  # 复用前面的 FM 层

        # Deep: MLP
        in_dim = num_fields * embed_dim
        layers = []
        for h in hidden_units:
            layers += [nn.Linear(in_dim, h), nn.ReLU(), nn.Dropout(0.1)]
            in_dim = h
        layers.append(nn.Linear(in_dim, 1))
        self.deep = nn.Sequential(*layers)
        self.bias = nn.Parameter(torch.zeros(1))

    def forward(self, x):
        x_offset = x + self.offsets.to(x.device)
        embed = self.embedding(x_offset)           # (batch, fields, k) — 共享!
        linear_out = self.linear(x_offset).sum(1)  # (batch, 1)
        fm_out = self.fm(embed)                     # (batch, 1)
        deep_out = self.deep(embed.view(x.size(0), -1))
        logit = self.bias + linear_out + fm_out + deep_out
        return torch.sigmoid(logit.squeeze(1))

DIN 深度兴趣网络

Alibaba, KDD 2018 | 注意力机制引入推荐
核心洞察：用户兴趣是多样的、局部的。对于不同的候选广告，应该激活不同的历史行为子集。DIN 通过 Local Activation Unit (注意力机制) 让用户表示随候选物品动态变化，打破固定长度向量的表达瓶颈。

问题：固定长度表示的局限

在 DIN 之前，用户行为序列通过 Sum/Mean Pooling 压缩为固定向量。一个同时对运动鞋、电子产品和书籍感兴趣的用户，在预测运动鞋点击率时，电子产品的行为就是噪声。固定向量被迫编码所有兴趣，表达能力受限。

为什么不做 Softmax？

传统注意力 (Softmax)

权重归一化到 [0,1] 且和为 1。问题：当用户有 100 个历史行为，但只有 2 个与候选相关时，相关行为的权重被 98 个不相关行为稀释。丢失了"兴趣强度"信息。

DIN (无 Softmax)

直接用激活函数输出作为权重。权重之和本身就编码了"整体兴趣强度"——和大说明用户对该类目兴趣浓厚，和小说明兴趣淡薄。保留了 intensity 信息。

Dice 激活函数

$$\text{Dice}(x) = p(x) \cdot x + (1 - p(x)) \cdot \alpha x, \quad p(x) = \frac{1}{1+e^{-\frac{x-E[x]}{\sqrt{Var[x]+\epsilon}}}}$$

本质是数据自适应版的 PReLU：转折点不是固定的 0，而是由 batch 统计量决定

Python (PyTorch) — Attention Unit
class AttentionUnit(nn.Module):
    """DIN Local Activation Unit — 不做 Softmax"""
    def __init__(self, embed_dim, hidden_units=[64, 32]):
        super().__init__()
        in_dim = embed_dim * 4  # [query, key, query-key, query*key]
        layers = []
        for h in hidden_units:
            layers += [nn.Linear(in_dim, h), nn.PReLU()]
            in_dim = h
        layers.append(nn.Linear(in_dim, 1))
        self.mlp = nn.Sequential(*layers)

    def forward(self, query, keys, keys_mask=None):
        # query: (B, D) — 候选广告, keys: (B, T, D) — 用户行为序列
        query = query.unsqueeze(1).expand_as(keys)  # (B, T, D)
        att_input = torch.cat([query, keys, query - keys, query * keys], dim=-1)
        att_weight = self.mlp(att_input).squeeze(-1)  # (B, T)

        if keys_mask is not None:
            att_weight = att_weight.masked_fill(~keys_mask, -1e9)

        # 注意：没有 softmax! 直接作为权重
        att_weight = att_weight.unsqueeze(1)  # (B, 1, T)
        output = torch.bmm(att_weight, keys).squeeze(1)  # (B, D)
        return output

MMOE (Multi-gate Mixture-of-Experts)

Google, KDD 2018 | 专家混合解决负迁移
核心洞察：不同任务需要不同的特征表示。MMOE 用多个 Expert 网络学习不同的特征变换，每个任务通过自己的 Gate 网络自适应地选择和组合这些 Expert 的输出，从而在共享中保留任务个性。

MMOE 输出公式：

$$y_k = h_k\bigl(f_k(\mathbf{x})\bigr), \quad f_k(\mathbf{x}) = \sum_{i=1}^{n} g_k^{(i)}(\mathbf{x}) \cdot E_i(\mathbf{x})$$

\(g_k(\mathbf{x}) = \text{Softmax}(W_{gk} \cdot \mathbf{x})\) 是任务 k 的 Gate，\(E_i\) 是第 i 个 Expert

Python (PyTorch)
class MMOE(nn.Module):
    def __init__(self, input_dim, expert_dim, num_experts, num_tasks, tower_dims):
        super().__init__()
        self.experts = nn.ModuleList([
            nn.Sequential(nn.Linear(input_dim, expert_dim), nn.ReLU())
            for _ in range(num_experts)
        ])
        self.gates = nn.ModuleList([
            nn.Sequential(nn.Linear(input_dim, num_experts), nn.Softmax(dim=-1))
            for _ in range(num_tasks)
        ])
        self.towers = nn.ModuleList([
            nn.Sequential(nn.Linear(expert_dim, tower_dims), nn.ReLU(), nn.Linear(tower_dims, 1))
            for _ in range(num_tasks)
        ])

    def forward(self, x):
        expert_outs = [e(x) for e in self.experts]       # list of (B, D)
        expert_outs = torch.stack(expert_outs, dim=1)  # (B, num_experts, D)

        outputs = []
        for gate, tower in zip(self.gates, self.towers):
            gate_out = gate(x).unsqueeze(-1)            # (B, num_experts, 1)
            mixed = (expert_outs * gate_out).sum(dim=1) # (B, D)
            outputs.append(torch.sigmoid(tower(mixed)))
        return outputs

PLE (Progressive Layered Extraction)

Tencent, RecSys 2020 | 解决跷跷板现象
核心洞察：MMOE 的所有 Expert 都是共享的，当任务冲突严重时仍会产生"跷跷板"——优化一个任务会降低另一个。PLE 的解法：引入任务特定 Expert + 共享 Expert，并通过多级抽取 (Progressive Extraction) 逐层解耦。

PLE vs MMOE 核心区别

维度MMOEPLE
Expert 类型全部共享任务特定 + 共享
层级结构单层多级渐进抽取
Gate 输入仅看所有共享 Expert看本任务 Expert + 共享 Expert
任务冲突仍可能跷跷板显式隔离减轻跷跷板
参数量较少较多（但效果显著提升）

ESMM (Entire Space Multi-Task Model)

Alibaba, SIGIR 2018 | 全空间 CVR 建模
核心洞察：CVR 模型只能在点击样本上训练（样本选择偏差），但需要对全部曝光样本预测（推理空间不匹配）。ESMM 利用数学关系 pCTCVR = pCTR × pCVR，在全空间（所有曝光）上训练 CTCVR，从而绕过选择偏差问题。
$$p(\text{conversion}|\text{impression}) = \underbrace{p(\text{click}|\text{impression})}_{\text{pCTR (Tower 1)}} \times \underbrace{p(\text{conversion}|\text{click})}_{\text{pCVR (Tower 2)}}$$

Loss = CTR Loss (全空间, click label) + CTCVR Loss (全空间, conversion label)

ESMM 如何解决选择偏差

传统 CVR 训练

仅在点击样本上训练 → 学到的是 P(conv|click, 被点击的样本) → 推理时面对全部曝光样本，分布不匹配

ESMM

CTCVR 的 label 在全空间都有 → 通过乘法约束隐式训练 CVR → CVR Tower 从未直接看到选择偏差的 label

多任务模型选择指南

场景推荐模型原因
任务高度相关Shared-Bottom / MMOE共享充分，结构简单
任务相关性中等MMOEGate 自适应平衡共享与独立
任务有冲突PLE任务特定 Expert 隔离冲突
CTR → CVR 漏斗关系ESMM利用数学关系消除选择偏差
多目标 + 个性化PLE + PEPNetPLE 解耦任务 + PEPNet 个性化参数

PEPNet (Parameter & Embedding Personalized Net)

Kuaishou, KDD 2023 | 个性化参数网络
核心洞察：传统模型所有用户共享同一套网络参数，而用户偏好千差万别。PEPNet 通过两级个性化——EPNet (Embedding 级) 和 PPNet (参数级)——让网络参数根据每个用户的先验信息动态调整，实现"千人千模"的效果。

EPNet vs PPNet

组件作用层级机制效果
EPNetEmbedding 层根据先验信息生成 Gate 向量，逐元素缩放特征 Embedding在特征表示阶段就注入个性化信号
PPNetDNN 每一层根据先验信息为 DNN 每一层的输出生成缩放/偏移让网络参数行为因人而异

EPNet:

$$\mathbf{e}'_i = \sigma\bigl(\text{MLP}_{\text{EP}}(\text{prior})\bigr)_i \odot \mathbf{e}_i$$

PPNet (每层):

$$\mathbf{h}'_l = \sigma\bigl(\text{MLP}_{\text{PP}}^{(l)}(\text{prior})\bigr) \odot \mathbf{h}_l$$

\(\sigma\) = Sigmoid，\(\odot\) = 逐元素乘法，prior = 用户/场景先验特征

Python (PyTorch)
class EPNet(nn.Module):
    def __init__(self, prior_dim, num_fields, embed_dim):
        super().__init__()
        gate_dim = num_fields * embed_dim
        self.gate = nn.Sequential(
            nn.Linear(prior_dim, gate_dim), nn.ReLU(),
            nn.Linear(gate_dim, gate_dim), nn.Sigmoid()
        )

    def forward(self, embeddings, prior):
        # embeddings: (B, F*D), prior: (B, P)
        gate = self.gate(prior)           # (B, F*D)
        return embeddings * gate            # 逐元素缩放

class PPNet(nn.Module):
    def __init__(self, prior_dim, hidden_dims):
        super().__init__()
        self.gates = nn.ModuleList([
            nn.Sequential(nn.Linear(prior_dim, h), nn.Sigmoid())
            for h in hidden_dims
        ])

    def forward(self, hidden_outputs, prior):
        # hidden_outputs: list of (B, H_l)
        results = []
        for h, gate_fn in zip(hidden_outputs, self.gates):
            gate = gate_fn(prior)
            results.append(h * gate)      # 参数级个性化
        return results

DCN: Deep & Cross Network

Google, AdKDD 2017 (v1) / WWW 2021 (v2)
核心洞察：DNN 学习特征交叉是隐式的、低效的 (bit-wise)。DCN 引入 Cross Network，通过简单的递推结构显式且高效地学习有界多项式阶的特征交叉 (vector-wise)。DCN-V2 将秩-1 矩阵升级为满秩矩阵，并用 MoLR 控制成本。

Cross Network 递推公式

DCN v1 (Rank-1):

$$\mathbf{x}_{l+1} = \mathbf{x}_0 \mathbf{x}_l^\top \mathbf{w}_l + \mathbf{b}_l + \mathbf{x}_l$$

参数量: \(d \times L\)（每层仅一个 \(d\) 维向量 \(\mathbf{w}_l\)）

DCN v2 (Full-Rank):

$$\mathbf{x}_{l+1} = \mathbf{x}_0 \odot (\mathbf{W}_l \mathbf{x}_l + \mathbf{b}_l) + \mathbf{x}_l$$

参数量: \(d^2 \times L\)（每层一个 \(d \times d\) 矩阵，用 MoLR 压缩：\(W = \sum_{i=1}^R U_i V_i^\top\)）

DCN v1 (Rank-1)
DCN v2 (Full-Rank + MoLR)

优势

极少参数 (d×L)，高效，显式交叉到 L+1 阶

局限

\(\mathbf{x}_0 \mathbf{x}_l^\top\) 是秩-1 矩阵，表达能力受限，每层只能学习一种交叉模式

改进

满秩 \(W_l\) 矩阵，表达能力大幅提升。MoLR 分解：\(W = \sum_{i=1}^R U_i V_i^\top\)，R=64 时接近满秩效果但参数量可控

Stacked vs Parallel

Stacked: Cross → Deep 串行，Cross 输出喂给 Deep
Parallel: Cross ∥ Deep 并行，输出拼接。实验显示 Stacked 通常更优

Python (PyTorch) — Cross Network
class CrossNetworkV2(nn.Module):
    """DCN-V2 Cross Network: 满秩交叉 + 可选 MoLR"""
    def __init__(self, in_dim, num_layers, low_rank=None):
        super().__init__()
        self.num_layers = num_layers
        if low_rank:  # MoLR: W = U @ V.T
            self.U = nn.ParameterList([nn.Parameter(torch.randn(in_dim, low_rank)) for _ in range(num_layers)])
            self.V = nn.ParameterList([nn.Parameter(torch.randn(in_dim, low_rank)) for _ in range(num_layers)])
        else:  # Full-rank W
            self.W = nn.ParameterList([nn.Parameter(torch.randn(in_dim, in_dim)) for _ in range(num_layers)])
        self.bias = nn.ParameterList([nn.Parameter(torch.zeros(in_dim)) for _ in range(num_layers)])
        self.low_rank = low_rank

    def forward(self, x0, x=None):
        if x is None: x = x0
        for i in range(self.num_layers):
            if self.low_rank:
                xw = x @ self.V[i] @ self.U[i].T  # MoLR
            else:
                xw = x @ self.W[i]
            x = x0 * (xw + self.bias[i]) + x  # 残差连接
        return x
专题
08

CVR 建模的特殊挑战与解法

工业实践中最复杂的建模问题之一
核心问题：CVR 和 CTR 在形式上都是二分类，但 CVR 面临五大额外挑战：样本选择偏差、数据稀疏、延迟转化、归因窗口、预估校准。直接把 CTR 的架构套到 CVR 上一定会失败。

CVR vs CTR 核心区别

维度CTR 建模CVR 建模
训练样本全部曝光样本仅点击样本 (选择偏差)
正样本比例通常 2-10%通常 0.1-5% (极度稀疏)
标签时效点击即时发生转化可能延迟数小时/天
推理空间全部曝光样本也需要对全部曝光预测
校准要求中等 (排序为主)极高 (直接用于出价)

五大挑战及解法

挑战 1: 样本选择偏差 (Sample Selection Bias)

CVR 模型在点击样本上训练，但需要对全空间预测。点击用户是有偏子集——他们天然对该物品更感兴趣。

解法：ESMM (pCTCVR = pCTR × pCVR)、因果推断方法 (IPW 加权)

挑战 2: 数据稀疏 (Data Sparsity)

转化事件比点击事件少 10-100 倍，正样本极度稀缺。

解法：多任务学习共享 CTR 的样本信号、预训练 + 微调、数据增强

挑战 3: 延迟转化 (Delayed Conversion)

用户可能在点击后数小时甚至数天才完成转化。训练时使用的 label 可能是错误的（假阴性）。

解法：Delayed Feedback Model (DFM)、ESDF、等待窗口 + 假阴性校准

挑战 4: 归因窗口 (Attribution)

用户可能通过多次点击不同广告后才转化。哪次点击应该获得转化功劳？

解法：Last-click、Time-decay、Data-driven attribution (Shapley值)

挑战 5: 预估校准 (Calibration)

CVR 预估值直接参与出价 (bid = target_CPA × pCVR)，需要绝对值准确，而非仅排序准确。

解法：Platt Scaling、Isotonic Regression、Field-aware Calibration

Platt Scaling 校准：

$$p_{\text{cal}} = \sigma(a \cdot \text{logit} + b), \quad a, b = \arg\min -\sum_i [y_i \log p_i + (1-y_i)\log(1-p_i)]$$

在验证集上学习 a, b 两个参数，将模型输出校准为真实概率

CVR 在广告系统中的特殊地位

$$\text{eCPM} = \text{bid} \times \text{pCTR} = \text{target\_CPA} \times \text{pCVR} \times \text{pCTR}$$

pCVR 偏高 → 广告主超支 → 流失；pCVR 偏低 → 平台收入损失
